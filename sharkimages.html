Why?

I developed this site to learn how images pass over inter-network and gets stored in a remote server. This is a simple platform to upload your images and recoup them back when you need from the website through internet.

Technologies used

Backend : python-Flask
Frontend : Html, Css, Twitter Bootstrap, JavaScript

How to use it?


It's very simple, you just have to either drag and drop or select the picture you want to upload and press UPLOAD button, that's it. Find your uploaded pictures in EXPLORE.


Supposedly people want to keep their personal pictures private, SharkImages got solution for that too! Just create an account in SharkImages and upload your pictures. If you want a picture to be publicised permanently then press 'MAKE IT PUBLIC' button.

Project log


  • First day there was just a file selector, an upload button and upload confirmation page with share url for the image.

  • Drag and drop feature was implemented using javascript in the next stage.

  • Storing images in database was my initial plan. I searched alot for a python module that could do database operations of images for me, couldn't find it!! Then I wrote it on my own( SPIDERSQLITE3 ). Later I changed the storage to file system to increase smoothness of the website.

  • The must feature 'explore' was the next move, where you can find the uploaded images.

  • Considering people to have their private images in SharkImages, I developed user account login system using flask session,

  • Added bootstrap navigation bar.

  • Using Bootstrap modal form created popup modal for login and registration forms.
